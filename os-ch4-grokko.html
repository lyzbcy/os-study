01学习目标与阅读路径

本章对应考试 1 道大题，三选一出题：磁盘调度算法（算移臂距离）、记录优化分布（算旋转等待）、位示图（算字号位号）。复习时间约 2 小时。

目标 1 · 磁盘调度
给定磁头位置和请求序列，能用 FCFS / SSTF / SCAN / CSCAN 四种算法分别算出服务顺序和总移动距离。
目标 2 · 记录优化分布
给定盘转速、扇区数、读时间、处理时间，能算出优化分布的间隔扇区数和优化前后的总时间。

02零基础背景包

学过但生疏？下面 5 个概念用一句话唤醒。

背景补充 ① · 磁盘物理结构（唤醒）
磁盘有多个盘面，每面有同心圆磁道，磁道上一段弧是扇区。所有盘面上同一半径的磁道组成一个柱面。
一句话记忆：柱面是"一圈盘子在同一半径上的所有磁道叠成的圆柱"。
物理地址三元组：磁头号（盘面号）+ 磁道号（柱面号）+ 扇区号。
背景补充 ② · 一次访盘时间（唤醒）
一次访盘时间 = 寻道时间 + 旋转延迟时间 + 存取时间
寻道=磁头移动到目标柱面（最慢，调度算法主要优化它）；旋转延迟=盘片转到目标扇区；存取=读写数据。
背景补充 ③ · FCB 与 I节点（唤醒）
FCB（文件控制块）= OS 管理文件的数据结构，FCB 存在 = 文件存在。I节点=把 FCB 里的描述信息单独抽出来，目录项只留"文件名 + I节点号"，优点是便于共享、加快检索。
背景补充 ④ · 位示图（唤醒）
位示图=用一位二进制（0/1）表示一个磁盘块的空闲/已分配状态。整个磁盘的状态压缩成一串 01。关键操作是字号位号与块号的换算。
背景补充 ⑤ · 记录优化分布（唤醒）
磁盘转着圈，处理一个记录需要时间。如果记录紧挨着放，处理完上一个，下一个已经转过去了，得等一整圈。优化分布 = 精确算出间隔，让下一个记录正好转到磁头下。

05核心判断与依据

事实
判断 1：调度算法主要优化寻道时间
寻道时间（磁头移动）远大于旋转延迟和存取时间，所以四种调度算法的核心都是减少磁头移动距离。
原文依据：「磁盘调度的目标：减少寻道时间，减少延迟时间」
判断
判断 2：SSTF 可能饥饿，SCAN/CSCAN 不会
SSTF 选最近的请求，如果持续有更近的新请求到来，远处请求会一直得不到服务。SCAN（电梯）和 CSCAN 会扫过所有柱面，不会饥饿。
原文依据：「SSTF 改善平均时间，但造成某些访问请求长期等待得不到服务」「SCAN 到头再反向」
判断
判断 3：记录优化分布的间隔由"读时间+处理时间"决定
要让处理完上一个记录时下一个正好转到磁头下，间隔扇区数 = (读时间 + 处理时间) ÷ 每扇区旋转时间。
原文依据：「优化要求前一个记录处理完，下一个记录正在磁头下」

06关键概念拆解

四种磁盘调度算法对比
算法核心规则优缺点
FCFS按请求到达顺序服务简单公平，但可能最内到最外反复移动
SSTF选距当前磁头最近的请求改善平均时间，但可能饥饿
SCAN（电梯）一个方向移到头，再反向不会饥饿，需判断初始方向
CSCAN（单向）总从 0 向里扫到最大，快速返回 0返回时不服务
柱面（Cylinder）
所有盘面上相同磁道号（相同半径）的磁道组成的集合。磁头移动以柱面为单位计算寻道距离。
不要误解成：柱面不是单个磁道，是"一圈盘面在同一半径上叠成的圆柱"。
位示图换算公式
关键：看清题里"行列从 0 还是 1 编号"。
若从 0 编号、字长 W：
• 字号 = 块号 ÷ W（整除）
• 位号 = 块号 mod W（取余）
若从 1 编号，需要 +1 或 −1 调整。
原文依据：位示图管理磁盘空间，每块 1 位。

08手把手计算演示

本章两大计算题型，每道从读题到答案每一步都写出来。

演示 1 · 磁盘调度四算法（FCFS / SSTF / SCAN）

大题标准题型：给定磁头位置和请求序列，分别算各算法的服务顺序和总移动距离。

已知：磁盘有 200 个柱面（0~199），磁头当前在 53 号柱面，向柱面号减小方向移动。
请求序列（按到达顺序）：98, 183, 37, 122, 14, 124, 65, 67。
求：FCFS、SSTF、SCAN 三种算法的服务顺序和总移动距离。
第 1 步 · FCFS（按到达顺序服务）
服务顺序就是请求顺序：53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67
移动距离逐段算：
|98−53|=45，|183−98|=85，|37−183|=146，|122−37|=85，|14−122|=108，|124−14|=110，|65−124|=59，|67−65|=2
总移动 = 45+85+146+85+108+110+59+2 = 640 个柱面
第 2 步 · SSTF（每次选最近的）
从 53 出发，找最近的请求：
• 53 → 65（|65−53|=12，最近）
• 65 → 67（|67−65|=2）
• 67 → 37（|37−67|=30，比 98 近）
• 37 → 14（|14−37|=23）
• 14 → 98（|98−14|=84，比 122 近）
• 98 → 122（|122−98|=24）
• 122 → 124（|124−122|=2）
• 124 → 183（|183−124|=59）
服务顺序：53→65→67→37→14→98→122→124→183
总移动 = 12+2+30+23+84+24+2+59 = 236 个柱面（比 FCFS 少很多）
第 3 步 · SCAN（电梯算法，向减小方向先走到底）
磁头当前向减小方向移动，所以先把比 53 小的请求按从大到小服务，到底（0）后反向服务比 53 大的：
• 向小走：53 → 37 → 14（到底，反向）
• 反向向大走：14 → 65 → 67 → 98 → 122 → 124 → 183
服务顺序：53→37→14→65→67→98→122→124→183
移动：到 14 走了 53−14=39，再到 183 走了 183−14=169
总移动 = 39 + 169 = 208 个柱面
答：FCFS=640，SSTF=236，SCAN=208（柱面）。
口诀：FCFS 按序、SSTF 选近、SCAN 看方向走到底再反向。

演示 1b · CSCAN 单向扫描 & LOOK 电梯改进（真题考点）

演示 1 只画了 FCFS/SSTF/SCAN。但去年真题考了 LOOK，CSCAN 也常考。这俩和 SCAN 的区别是关键考点。

同样数据：磁头在 53，向磁道号减小方向移动，队列 98,183,37,122,14,124,65,67。
分别用 CSCAN 和 LOOK 算服务顺序和总移动量。
第 1 步 · CSCAN（单向扫描，到端点快速回起点不服务）
向减小方向扫到 0（端点），快速返回 199 端点（返回时不服务），再向减小扫：
53 → 37 → 14（到减小端点 0 附近）→ 快速返回 199 → 183 → 124 → 98 → 67 → 65
移动：53→37(16) → 14(23) → 到0再回199共 14+199=213 → 183(16) → 124(59) → 98(26) → 67(31) → 65(2)
总移动 ≈ 16+23+213+16+59+26+31+2 = 386
关键CSCAN 到端点后快速返回另一端，返回沿途不服务任何请求。
第 2 步 · LOOK（SCAN 改进：到最后一个请求即反向，不到物理端点）
LOOK vs SCAN 区别：SCAN 走到磁盘物理边界（0 或 199），LOOK 只走到该方向上最远的请求就立即折返，省去空跑。
向减小方向：先服务 <53 的请求（从大到小）：37, 14 → 到 14（减小方向最远请求）折返
再服务 >53 的请求（从小到大）：65, 67, 98, 122, 124, 183
完整顺序：53 → 37 → 14 → 65 → 67 → 98 → 122 → 124 → 183
移动：53→37(16) → 14(23)，折返 14→183(169)
总移动 = 16 + 23 + 169 = 208
第 3 步 · 四算法横向对比
算法到端点吗返回时服务吗本例总移动
SCAN走到物理端点(0/199)反向时服务约 208（同方向）
LOOK只到最远请求即折返反向时服务208
CSCAN走到物理端点返回时不服务386
记忆SCAN 像电梯"走到底再回"；LOOK 是"到最后一名乘客就回"（更省）；CSCAN 是"单行道，到头空车回起点"。
答：CSCAN 总移动约 386（到端点空车返回不服务）；LOOK 总移动 208（到最远请求即折返，不到物理端点）。
核心：LOOK 是 SCAN 的改进——不到边界、到最远请求就回，所以更省寻道时间。

演示 2 · 记录优化分布（旋转调度）

另一类大题：算顺序存放的总时间 vs 优化分布后的总时间。

已知：盘转一周 20ms，一个磁道分 10 个扇区，存放 10 个记录 A~J。
读一个记录 2ms，处理一个记录 4ms。记录按 A,B,C…顺序存放在相邻扇区。
求：(1) 顺序处理完所有记录的总时间；(2) 优化分布方案和优化后总时间。
第 1 步 · 算每扇区旋转时间
盘转一周 20ms，10 个扇区 → 每转过一个扇区 = 20 ÷ 10 = 2ms（正好等于读一个记录的时间）。
第 2 步 · 分析顺序存放的悲剧
• 读 A：2ms（磁头在 A 上）
• 处理 A：4ms。此时磁头已经转过了 4÷2=2 个扇区（B、C 都过去了！）
• 要读 B，B 已转走，必须等盘转完一整圈（20ms）再回到 B。
• 所以除最后一个记录外，每个记录花费 = 读2 + 处理4 + 等20 − (已转的) ，实际每个约 6+14=20ms 等待下一圈
• 具体计算：第 1~9 个记录每个花费约 22ms（含等下一圈），第 10 个只需 6ms（不用等）
总时间 ≈ 204ms（9×22 + 6 = 204，或按 6+9×22 等不同教材算法，约 200ms 量级）
第 3 步 · 算优化间隔
要让处理完上一个记录时下一个正好转到磁头下：
优化间隔扇区数 = (读时间 + 处理时间) ÷ 每扇区时间 = (2 + 4) ÷ 2 = 3 个扇区
意思是：A 放第 1 扇区，下一个记录 隔 3 个扇区放（第 4 扇区），再下一个放第 7 扇区……
第 4 步 · 写出优化分布方案并算总时间
优化分布：扇区 1=A, 4=B, 7=C, 10=D, 3=E, 6=F, 9=G, 2=H, 5=I, 8=J（每隔 3 个扇区放一个）。
优化后：处理完 A（6ms），B 正好转到磁头下，不用等！
每个记录花费 = 读 2 + 处理 4 = 6ms，无等待。
总时间 = 10 × 6 = 60ms
答：顺序存放约 204ms；优化分布（每隔 3 扇区放一个）后仅 60ms。
口诀：间隔 = (读+处理) ÷ 每扇区时间；优化后每个记录 6ms 无等待。

09误解与边界

误解 1：SCAN 不需要判断初始方向
大题扣分点SCAN 题必须看清磁头当前向哪个方向移动（向内/向外），方向不同服务顺序完全不同。
原文：「磁头按一个方向移动，过程中对遇到的请求服务」——方向是前提。
误解 2：CSCAN 返回时也服务请求
事实错CSCAN 从最大柱面返回 0 号是"快速返回"，不服务任何请求。返回后才重新从 0 向里扫描。
原文：「返回时不为任何的等待访问者服务」。
误解 3：SSTF 一定最优
推论错SSTF 只保证局部最优（每次选最近的），不保证全局最优，而且可能饥饿。
原文：「SSTF 改善平均时间，但造成某些访问请求长期等待」。
误解 4：记录优化分布就是随机散开
事实错不是随机散开，是精确计算间隔，让处理完上一个时下一个正好到磁头下。间隔 = (读+处理)÷每扇区时间。
误解 5：位示图行列编号不影响计算
事实错从 1 编号和从 0 编号的换算公式不同，必须看清题目条件再套公式。

10背景知识缺口
