01学习目标与阅读路径

本章对应考试 2 道大题：一道考地址变换（给页表/段表和逻辑地址，算物理地址），另一道考页面置换（给访问序列和块数，画表数缺页次数）。还会考可变分区分配和位示图换算。

目标 1 · 地址变换
给定页表和逻辑地址，3 步内算出物理地址（求页号→查页表→拼物理地址）。
目标 2 · 段式变换+越界判断
给定段表和逻辑地址（段号, 段内地址），算物理地址并判断是否越界中断。
目标 3 · 页面置换
能画出 FIFO / LRU / OPT 三种算法的置换表，数出缺页次数和缺页率。
目标 4 · 可变分区+位示图
能做首次/最佳/最坏适应的分配过程；能做位示图"第 N 块对应何字何位"的换算。

阅读路径：模块 2 唤醒基础概念 → 模块 3-6 过概念 → 模块 8 是重点，三道题手把手算 → 模块 9 避坑。

02零基础背景包

学过但生疏？下面 5 个概念用一句话唤醒。

背景补充 ① · 逻辑地址 vs 物理地址（唤醒）
逻辑地址是程序编译后产生的相对地址（从 0 开始，虚的）；物理地址是 CPU 实际访问内存的绝对地址（实实在在的内存单元号）。OS 的工作就是把逻辑地址翻译成物理地址——这叫地址变换/重定位。
一句话记忆：逻辑地址是书页上的"第几页第几行"，物理地址是这本书在书架上的绝对位置。
背景补充 ② · 页 vs 块（唤醒）
分页管理里，逻辑地址空间按固定大小切成一个个页（page），物理内存按同样大小切成一个个块/页框（frame）。页和块大小相同，通过页表建立映射：页号 → 块号。
一句话记忆：页是逻辑的格子，块是物理的格子，页表就是"哪个逻辑格子放进哪个物理格子"的对照表。
背景补充 ③ · 局部性原理（唤醒）
程序执行时访问的地址集中在小范围内。时间局部性：刚用过的地址近期还会用（所以 LRU 才有效）；空间局部性：用了某个地址，附近的也会被用（所以按页调入才划算）。
一句话记忆：程序不是乱跑的，它会在一个小区域里磨蹭很久。
背景补充 ④ · 缺页中断（唤醒）
CPU 访问的页不在内存里时，硬件产生缺页中断，OS 从外存把该页调入内存；如果内存已满，就用置换算法淘汰一页再调入。
一句话记忆：缺页中断 = 发现要的东西不在手边，得去仓库取。
背景补充 ⑤ · Belady 现象（唤醒）
一般直觉是"给的块越多缺页越少"，但 FIFO 算法可能出现反常：块数增加，缺页反而增加。这就是 Belady 现象。LRU 和 OPT 不会出现。
一句话记忆：Belady = 给多了反而更糟，只发生在 FIFO。

03原文问题定位

表面问题 这章在讲什么？
内存管理的各种机制：连续分配（分区）、非连续分配（分页/分段）、虚拟内存（置换算法）。
深层问题 它在回答什么？
两个核心问题：
① 有限的物理内存怎么分给多个程序？（分配方式）
② 程序太大、内存装不下怎么办？（虚拟内存：请求调入 + 置换）
考试考查的就是这两个问题的具体操作：地址怎么转换（大题1）+ 页面怎么换（大题2）。
事实原文：「虚拟存储器：请求调入功能和置换功能，能从逻辑上对内存容量加以扩充」「逻辑容量由内存容量和外存容量之和决定」

05核心判断与依据

事实
判断 1：页式地址变换 = 求页号 → 查页表 → 拼物理地址
页内偏移在逻辑地址和物理地址里完全相同，因为页和块大小一样。需要查表变换的只是页号 → 块号。
原文依据：「页式存储管理采用动态重定位；逻辑地址 = 页号 + 页内地址；查页表得块号；物理地址 = 块号 × 页大小 + 页内偏移」
事实
判断 2：段式变换必须先判断越界
段长是可变的，所以段式地址变换必须先检查段内地址 < 段长，否则产生越界中断。这是段式与页式的关键区别（页式不需要越界判断，因为页大小固定）。
原文依据：「在分段系统中进行地址转换时，先将段号与段表长度比较……然后再将段内地址和段长作比较，若不越界则相加」
判断
判断 3：LRU 性能接近 OPT，FIFO 最差且可能 Belady
OPT 是理论最优（不可实现），LRU 基于局部性原理接近 OPT，FIFO 最简单但性能差且可能出现 Belady 现象。LRU 和 OPT 不会出现 Belady（它们是栈式算法）。
原文依据：「FIFO 会产生 Belady 现象；LRU 淘汰最近最久未访问的页（基于局部性）；OPT 是衡量标准」
事实
判断 4：虚拟内存容量 = 内存 + 外存
虚拟存储器通过请求调入 + 置换，逻辑上扩充内存。逻辑容量由内存容量和外存容量之和决定，速度接近内存，成本接近外存。
原文依据：「其逻辑容量由内存容量和外存容量之和所决定」

06关键概念拆解

页表（Page Table）
原文：记录页号与块号对应关系的数据结构。
人话：一本"哪个逻辑页放在哪个物理块"的对照册。页内偏移不用查表，直接搬过去。
关键：有了页表，逻辑地址就能翻译成物理地址；为了加速查表，引入快表 TLB（页表的高速缓存）。
快表 TLB
原文：存放在高速缓冲寄存器中的页表叫快表，这个寄存器叫联想存储器。
人话：页表的"快捷方式"。没有 TLB 时每次访存要查两次（查页表 + 取数据）；有 TLB 命中就只需一次。
原文依据：「执行一次访内操作至少要访问主存两次……为了提高存取速度，设置高速缓冲寄存器存放页表的一部分」
位示图（Bitmap）
原文：一个系统只有一张存储页面表，指出内存各页面是否被分配。
人话：用一位二进制表示一个块的状态（0 空闲、1 已分配）。整个内存的状态压缩成一串 01。
关键换算：第 N 块 → 字号 = N div 字长，位号 = N mod 字长（注意题里编号从 0 还是从 1 开始）。
抖动（Thrashing）与工作集
抖动：页面在内存和外存之间频繁换入换出，CPU 大量时间浪费在调页上。
原因：分配给进程的物理页架过少 / 淘汰算法不合理。
工作集：进程在某段时间内访问的页面集合。分配页架数 < 工作集大小就会抖动。
原文依据：「如果在一段时间内，作业占用的主存块数目小于工作集时，运行过程中就会不断出现缺页中断，从而导致系统的抖动」
四种页面置换算法对比
算法淘汰谁特点
OPT未来最久不访问的页理论最优不可实现，作衡量标准
FIFO最先调入内存的页简单，性能差，会出现 Belady
LRU最近最久未访问的页基于局部性，性能好，实现复杂
CLOCK引用位为 0 的页LRU 的近似实现，硬件支持简单

08手把手计算演示

本章三大计算题型，每道题从读题到答案每一步都写出来。看懂这三道，大题稳了一半。

演示 1 · 页式地址变换（3 步法）

大题1标准题型：给页表和逻辑地址，算物理地址。套路固定，背熟三步。

已知：页面大小 1024B，页表：0→2, 1→3, 2→1, 3→6（页号→块号）。
求：逻辑地址 2148 对应的物理地址。
第 1 步 · 拆逻辑地址 → 求页号 P 和页内偏移 W
页号 P = ⌊逻辑地址 ÷ 页面大小⌋
页内偏移 W = 逻辑地址 mod 页面大小
P = ⌊2148 ÷ 1024⌋ = ⌊2.099...⌋ = 2
W = 2148 − 2 × 1024 = 2148 − 2048 = 100
第 2 步 · 查页表：页号 → 块号
页表里"页号 2 → 块号 1"，所以 F = 1。
关键页号超出页表长度会"地址越界中断"（如本题页号 4 超出最大页号 3，就非法）。
第 3 步 · 拼物理地址
物理地址 = 块号 F × 页面大小 + 页内偏移 W
物理地址 = 1 × 1024 + 100 = 1124
✓ 答：逻辑地址 2148 → 物理地址 1124。
（页内偏移 100 直接搬过去，查表只换了页号 2→块号 1）

演示 2 · 页面置换表（FIFO vs LRU，同一序列对比）

大题2标准题型：给访问序列和块数，画表数缺页。这里用 3 个块、一个短序列对比 FIFO 和 LRU。

已知：内存 3 块，访问序列：1, 2, 3, 4, 1, 2, 5。
开始内存为空，首次访问必缺页。
分别用 FIFO 和 LRU 求缺页次数。
第 1 步 · 理解 FIFO（先进先出）
FIFO = 淘汰最先调入的那页（队列，先进先出）。
画表时：每来一页，若内存有就命中（不缺页）；若没有就缺页，内存满就淘汰最早进来的。
第 2 步 · 画 FIFO 表
访问1234125
块11114445
块2222111
块333322
缺页?是是是是是是是
第 4 步访问 4，内存{1,2,3}没有，淘汰最早进来的 1 → 换成 4。
第 5 步访问 1，内存{4,2,3}没有，淘汰最早的 2 → 换成 1（注意：FIFO 看的是调入顺序，不是最近访问）。
FIFO 缺页 = 7 次（本例全缺页）。
第 3 步 · 画 LRU 表（淘汰最近最久未访问）
访问1234125
块11114445
块2222111
块333322
本例 LRU 结果与 FIFO 相同（序列特殊），但通常不同。LRU 关键：淘汰时看"谁最久没被碰过"，刚被访问的页面优先级最高。
第 4 步 · 数缺页次数和缺页率
缺页次数 = 表里"缺页?是"的格子数。
缺页率 = 缺页次数 ÷ 访问总次数。
本例：7 次缺页 ÷ 7 次访问 = 100%。
注意题目说"前 4 页已在内存"时，前 4 次不算缺页，要从第 5 次开始数。
✓ 答：FIFO 缺页 7 次，LRU 缺页 7 次（本例特殊），缺页率 100%。
口诀：画表时每次访问后更新内存状态，缺页就标红数。

演示 3 · 可变分区分配（首次适应 vs 最佳适应）

选择题/小大题常考：给空闲区序列和作业请求，比较两种算法的分配结果。

已知：内存按地址顺序依次有 3 个空闲区：F1=32K、F2=10K、F3=228K。
现要依次分配作业 J1=20K、J2=10K。
分别用首次适应和最佳适应，说明每个作业被分到哪个空闲区。
第 1 步 · 理解三种算法
• 首次适应（First Fit）：按地址顺序找，第一个能放下的就分配。
• 最佳适应（Best Fit）：找所有能放下的里最小的那个（最接近需求尺寸，少浪费）。
• 最坏适应（Worst Fit）：找最大的空闲区分配（剩得多，将来还能用）。
第 2 步 · 首次适应分配过程
J1=20K：按地址找，F1=32K 能放下 → 分配，F1 剩 12K。
J2=10K：按地址找，F1 剩 12K 能放下 → 分配，F1 剩 2K。
结果：两个作业都挤在 F1，剩 2K 碎片。
第 3 步 · 最佳适应分配过程
J1=20K：能放下的有 F1=32K、F3=228K，选最小的 F1=32K → 分配，F1 剩 12K。
J2=10K：能放下的有 F1=12K、F2=10K、F3=228K，选最小的 F2=10K（正好！不浪费）→ 分配，F2 剩 0K。
结果：J1 在 F1，J2 在 F2，F2 零浪费。
第 4 步 · 对比结论
最佳适应倾向于"选最贴身的"，能减少大空闲区被切碎；但长期会产生很多极小的不可用碎片。
首次实现简单速度快，但可能在低地址区堆积大量小碎片。
考试常考首次适应分配后申请大块可能失败，最佳适应能成功（或反之）——具体看题目数据。
✓ 答：首次适应下 J1、J2 都进 F1（剩 2K）；最佳适应下 J1 进 F1、J2 进 F2（F2 零浪费）。
口诀：首次"看到能放就放"，最佳"挑最合身的"。

09误解与边界

误解 1：快表就是 Cache
易错快表（TLB）是页表项的高速缓存，不是通用数据缓存。它只缓存"页号→块号"的映射，加速地址变换。
原文：「存放在高速缓冲寄存器中的页表叫快表」。
误解 2：块数越多缺页一定越少
大题扣分点不对。FIFO 会出现 Belady 现象——块数增加缺页反而增加。只有 LRU 和 OPT 满足"栈式算法"性质，块数越多缺页单调不增。
原文：「FIFO 会产生 Belady 现象」。
误解 3：分页和分段是一回事
推论错页是固定大小自动划分（系统决定，用户无感）；段是按逻辑信息可变长划分（用户/编译器决定，如代码段、数据段）。所以段式必须查越界，页式不用。
原文：「分页：大小相同区域；分段：每个段定义一组逻辑信息，长度可变」。
误解 4：虚拟内存容量 = 内存大小
事实错虚拟内存逻辑容量 = 内存 + 外存，远大于物理内存。速度接近内存，成本接近外存。
原文：「其逻辑容量由内存容量和外存容量之和所决定」。
