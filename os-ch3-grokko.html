模块 2

零基础背景包

进入正文之前，必须先弄清以下基础概念，否则后面的地址变换和置换算法会完全看不懂。

背景补充 1：逻辑地址 vs 物理地址

程序编译后产生的是逻辑地址（相对地址/虚地址），从 0 开始编号。CPU 实际访问内存时用的是物理地址（绝对地址/实地址）。操作系统核心任务之一就是把逻辑地址翻译成物理地址——这就是"地址变换"或"重定位"。

背景补充 2：页面与页框

分页管理中，逻辑地址空间按固定大小切成一个个页（页面），物理内存也按同样大小切成一个个页框（物理块）。页和块大小相同，通过页表建立映射：页号→块号。

背景补充 3：局部性原理

程序执行时访问的地址往往集中在很小范围内。时间局部性：刚访问过的地址短期内还会访问。空间局部性：访问了某个地址，附近地址大概率也被访问。这是虚拟存储和 LRU 算法的理论基础。

背景补充 4：缺页中断

当 CPU 访问的页不在内存中时，硬件产生缺页中断，操作系统从外存把该页调入内存。如果内存已满，就需要用置换算法淘汰一页。

模块 5 · 大题核心

核心判断与依据：地址变换公式

这是考试必考的大题方向。必须记住以下公式并能熟练计算。

页式地址变换

页号 P = ⌊逻辑地址 / 页面大小⌋    页内地址 W = 逻辑地址 mod 页面大小
查页表得块号 F   →   物理地址 = 块号 F × 页面大小 + 页内地址 W

例题：页面大小 1024B，页表：0→2, 1→3, 2→1, 3→6。求逻辑地址 2148 的物理地址。

  • 页号 P = ⌊2148/1024⌋ = 2，页内地址 W = 2148 - 2×1024 = 100
  • 查页表：页号 2 → 块号 1
  • 物理地址 = 1 × 1024 + 100 = 1124

段式地址变换

段号 S + 段内地址 W → 查段表得段基址 B 和段长 L
若 W ≥ L → 越界中断    否则物理地址 = 段基址 B + 段内地址 W

例题：段表 [0] 基址 600K 段长 15KB。逻辑地址 [0, 137]：137 < 15KB 不越界，物理地址 = 600K + 137 = 615257。

有效存取时间

TAT = TLB命中率 × (查TLB + 访存) + (1 - TLB命中率) × (查TLB + 查页表 + 访存)
含缺页时再加缺页处理开销

2. 分页管理

定义：将逻辑地址空间分为固定大小的页，物理空间分为相同大小的块（页框），通过页表映射页号到块号。

地址变换：逻辑地址 = 页号 + 页内地址；查页表得块号；物理地址 = 块号 × 页大小 + 页内偏移。

不要误解：页大小和块大小相同，但页是逻辑概念，块是物理概念。

3. 分段管理

定义：按逻辑信息分段（代码段/数据段/栈段），每段连续分配但各段可不连续。

地址变换：逻辑地址 = 段号 + 段内地址；查段表得段基址；物理地址 = 段基址 + 段内地址。注意越界检查！

4. 页面置换算法

OPT：淘汰未来最久不访问的页，理论最优不可实现。

FIFO：淘汰最先调入的页，简单但可能产生 Belady 现象（块数增加缺页反而增加）。

LRU：淘汰最近最久未访问的页，基于局部性原理，性能好但实现复杂。

CLOCK：引用位扫描，1清0跳过，0淘汰，是 LRU 的近似实现。

5. Belady 现象

定义：分配的物理块数增加，缺页次数反而增加的现象。

出现于：FIFO 算法。

不出现于：LRU 和 OPT（它们满足栈式算法性质）。

模块 9

误解与边界

  • 误解：「快表就是 Cache」。实际：快表是页表项的高速缓存，不是通用数据缓存。
  • 误解：「块数越多缺页一定越少」。实际：FIFO 有 Belady 异常，块数增加可能反而增加缺页。
  • 误解：「分页和分段是一样的」。实际：页是固定大小自动划分，段是按逻辑信息可变长划分。
  • 误解：「虚拟内存容量等于内存大小」。实际：虚拟内存容量 = 内存 + 外存，远大于物理内存。
  • 误解：「抖动是因为算法太慢」。实际：抖动是物理页架不够导致频繁换页，解决方法是增加页架或改进算法。
