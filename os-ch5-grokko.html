02

零基础背景包

以下概念不是本章原文的观点，而是帮助你读懂本章的预备知识。

背景补充

CPU 与外设的速度差距

CPU 每秒执行数十亿条指令，而机械硬盘、打印机等外设的响应时间在毫秒级。一条 CPU 指令约 1 纳秒，一次磁盘 I/O 约 10 毫秒——速度差了 10⁷ 倍。

为什么现在需要知道：本章所有控制方式的演进，本质上都在解决这个速度不匹配的问题。

背景补充

中断（Interrupt）

CPU 正在执行程序时，硬件设备发出一个信号通知 CPU"某件事完成了"，CPU 暂停当前工作去处理这个通知，然后返回原来的程序。

为什么现在需要知道：中断方式是四种控制方式的第二种，也是理解 DMA 和通道的基础。

背景补充

总线（Bus）

计算机各部件之间的公共数据通道。CPU、内存、I/O 设备都连在总线上，谁使用总线谁就"占用"它。

为什么现在需要知道：DMA 的"周期窃取总线"就是在 CPU 不用总线的间隙，DMA 控制器偷偷使用总线传数据。

背景补充

独占 vs 共享

独占设备：同一时刻只能被一个进程使用（如打印机）。共享设备：多个进程可以交替使用（如磁盘）。独占设备如果分配不当会死锁。

为什么现在需要知道：SPOOLing 的核心价值就是把独占设备"改造"成共享设备。

05

核心判断与依据

事实 判断1：四种控制方式的并行程度递增

从轮询到中断到 DMA 到通道，CPU 从"全程盯着"变成"完全不管"，并行程度逐步提高。

为什么重要：这是整章的骨架，所有其他知识点都挂在这条演进线上。

原文依据：四种方式的演进——"并行程度递增（轮询→中断→DMA→通道）"

事实 判断2：轮询方式下 CPU 与外设完全串行

CPU 发出启动命令后必须循环测试设备状态，期间无法做其他事。不仅效率低，还无法发现和处理硬件错误。

可能的误读：不要以为"轮询就是定时检查"——轮询是持续循环测试，CPU 完全被占用。

原文依据："CPU 发启动命令后循环测试""CPU 和外设串行工作"

事实 判断3：DMA 实现块传输，CPU 仅首尾参与

DMA 控制器增加字节计数器和内存地址寄存器，在内存与 I/O 间直接成块传送。CPU 只在开始发"传送一块"命令和结束时处理。DMA 可周期窃取总线。

可能的误读："DMA 完全不需要 CPU"是错的。CPU 仍需启动和收尾。

原文依据："内存与 I/O 设备间直接成块传送""CPU 仅开始发命令，结束处理"

判断 判断4：SPOOLing 把独占设备改造成共享设备

SPOOLing 用大容量磁盘作缓冲，将慢速独占设备（如打印机）虚拟化，让多个进程"同时"使用。

原文依据："用大容量磁盘作缓冲，把独占设备改造成共享设备，I/O 与 CPU 并行"

事实 判断5：动态分配效率高但可能死锁

静态分配在运行前完成，利用率低但安全。动态分配在运行中按需分配，效率好但可能死锁。

原文依据："静态=运行前分配结束收回（利用率低）；动态=运行中按需分配停用即收（效率好但可能死锁）"

关键概念拆解

概念1：程序直接控制（轮询）
定义用户程序直接控制 I/O 传送，CPU 发启动命令后循环测试设备状态
人话CPU 全程守着设备，反复问"好了没？好了没？"
不要误解不是"定时检查"，是"持续循环测试"
原文依据："用户程序直接控制传送，CPU 发启动命令后循环测试"
概念2：中断方式
定义设备完成后主动发中断请求，CPU 响应并唤醒等待进程
人话CPU 不再傻等，设备好了会主动"敲门"通知
缺点每次 I/O 都要 CPU 干预，多设备时利用率仍会下降
原文依据："控制寄存器有中断位，设备完成后发中断请求，CPU 响应唤醒等待进程"
概念3：DMA 方式
定义内存与 I/O 设备间直接成块传送，DMA 控制器有字节计数器+内存地址寄存器
人话派一个"搬运工"（DMA 控制器）整批搬数据，CPU 只管开头下令和结尾收货
不要误解DMA 不是完全不需要 CPU，CPU 仍需首尾参与
原文依据："内存与 I/O 设备间直接成块传送""DMA 可周期窃取总线"
概念4：通道方式
定义独立于 CPU 的专门处理机，统一管理外设，实现 I/O 与 CPU 并行
人话雇一个"专职经理"（通道），CPU 把整个 I/O 任务外包出去
与 DMA 区别DMA 只管一块数据的传送；通道能管理多台外设的完整 I/O 流程
原文依据："独立于 CPU 的专门处理机，统一管理外设，实现 I/O 与 CPU 并行"
概念5：SPOOLing（假脱机操作）
定义Simultaneous Peripheral Operation On Line，用磁盘作缓冲把独占设备改造成共享设备
三组成① 输入井/输出井（磁盘） ② 输入缓冲区/输出缓冲区（内存） ③ 输入进程/输出进程
三特点① 提高 I/O 速度 ② 独占→共享 ③ 实现虚拟设备
不要误解SPOOLing ≠ 简单缓冲。它用磁盘模拟多台设备，是系统级的虚拟化方案
原文依据："用大容量磁盘作缓冲，把独占设备改造成共享设备"
概念6：设备分配四张表
DCT/DCB设备控制表 —— 记录每个设备的详细信息（设备级）
SDT系统设备表 —— 记录系统中所有设备（系统级总表）
COCT/COCB控制器控制表 —— 记录每个控制器信息（控制器级）
CHCT/CHCB通道控制表 —— 记录每个通道信息（通道级）
层级关系SDT → DCT → COCT → CHCT，从设备到通道逐级找到完整数据通路
原文依据："DCB/DCT 设备控制块；SDT 系统设备表；COCB/COCT 控制器控制块；CHCB/CHCT 通道控制块"

误解与边界

❌ 误解1："DMA 完全不需要 CPU 参与"

为什么容易误解： DMA 的核心卖点是"直接传送"，听起来像 CPU 完全不参与。

原文真正说了什么： CPU 仅开始发"传送一块"命令，结束时处理。中间的数据传输不需要 CPU，但首尾仍需 CPU。

原文没有说什么： 原文从没说 CPU 完全不参与 DMA 流程。

如何避免： 记住 DMA 的公式——"CPU 管开头和结尾，中间交给 DMA 控制器"。

❌ 误解2：SPOOLing 的输入井和输出井搞反

为什么容易误解： "输入"和"输出"的视角容易混淆——是从 CPU 角度还是从设备角度？

原文真正说了什么： 输入井是预输入程序把作业读到磁盘上（外设→磁盘，供 CPU 读取）；输出井是缓输出程序把数据写到打印机等设备（CPU→磁盘→外设）。

如何避免： 以"数据的流向"为准：输入井=数据从外面进来暂存的地方；输出井=数据要输出到外面之前暂存的地方。

原文依据："预输入程序读作业到输入井→主存装入作业→井管理程序读写→缓输出程序在打印机输出"

❌ 误解3："中断方式已经是最佳方案了"

为什么容易误解： 中断方式确实"大大提高 CPU 利用率"，听起来已经很好了。

原文真正说了什么： 中断方式的缺点是"每次 I/O 都要 CPU 干预，多设备时利用率仍降"。DMA 和通道进一步减少了 CPU 干预。

如何避免： 记住演进链——每种方式解决了前一种的缺点，没有"最好"只有"更好"。

❌ 误解4："SPOOLing 就是缓冲技术"

为什么容易误解： SPOOLing 确实用到了缓冲区。

原文真正说了什么： 缓冲区只是 SPOOLing 三组成之一。SPOOLing 是一个完整的系统——磁盘缓冲 + 内存缓冲 + 进程管理，实现独占→共享的虚拟化。

如何避免： SPOOLing ≠ Buffer。Buffer 只缓解速度差，SPOOLing 还改变了设备的使用方式（独占变共享）。
