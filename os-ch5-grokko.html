02零基础背景包

学过但生疏？下面 5 个概念用一句话唤醒。

背景补充 ① · CPU 与外设的速度差（唤醒）
CPU 每秒执行数十亿条指令，而打印机、机械硬盘等外设响应在毫秒级——速度差了 10⁷ 倍。
一句话记忆：本章所有 I/O 控制方式的演进，本质上都在解决"CPU 等外设太亏"这个问题。
背景补充 ② · 中断（唤醒）
CPU 正跑程序，外设发信号说"我做完了"，CPU 暂停当前工作去处理，处理完回来继续。
一句话记忆：中断 = 外设主动打电话告诉 CPU "我好了"，不用 CPU 一直傻等。
背景补充 ③ · 总线（唤醒）
计算机各部件的公共数据通道。CPU、内存、I/O 设备都连在总线上，谁用谁占。
一句话记忆：DMA 的"周期窃取"就是CPU 不用总线的间隙，DMA 控制器偷偷用。
背景补充 ④ · 独占 vs 共享设备（唤醒）
独占设备：同一时刻只能一个进程用（打印机，分配不当会死锁）。共享设备：多个进程交替用（磁盘）。虚拟设备=用 SPOOLing 把独占变共享。
背景补充 ⑤ · 缓冲区（唤醒）
两个速度不匹配的设备之间用一块内存暂存数据，弥补速度差。如 CPU（快）和打印机（慢）之间放缓冲区，CPU 写完就走，打印机慢慢取。

05核心判断与依据

事实
判断 1：四种 I/O 控制方式，并行程度递增
程序查询（CPU 与外设串行）→ 中断（CPU 与外设并行，但每传一个字要 CPU 干预）→ DMA（CPU 只管开始和结束，中间由 DMA 硬件成块传）→ 通道（通道是独立处理机，完全代替 CPU 管 I/O）。
原文依据：「主要差别在于中央处理器和外围设备并行工作的方式不同，并行工作的程度不同」
判断
判断 2：SPOOLing 把独占设备改造成共享设备
SPOOLing 用大容量磁盘作缓冲，把独占设备（打印机）改造成共享设备，实现 I/O 与 CPU 并行。输入井/输出井暂存多个进程的 I/O 请求，按队列依次服务。
原文依据：「SPOOLING 系统的特点：提高了 I/O 速度、将独占设备改造为共享设备、实现了虚拟设备功能」
事实
判断 3：设备分配需要四张表层层查找
设备分配按 DCB(设备) → COCB(控制器) → CHCB(通道) 层级查找，形成一条数据传输通路。SDT（系统设备表）是总入口。
原文依据：「数据结构：设备控制块 DCB、系统设备表 SDT、控制器控制块 COCB、通道控制块 CHCB」

06关键概念拆解

四种 I/O 控制方式对比
方式谁传数据CPU 参与度并行程度
程序查询（轮询）CPU全程（循环测试）最低（CPU 外设串行）
中断方式CPU间歇（每次中断参与）中（CPU 外设并行，但每字干预）
DMA 方式DMA 控制器少（仅首尾）较高（成块传，CPU 几乎不参与）
通道方式通道（独立处理机）极少（发指令即可）最高（通道完全代管）
SPOOLing（假脱机操作）
原文定义：Simultaneous Peripheral Operation On Line，用大容量磁盘作缓冲，把独占设备改造成共享设备。
三部分组成（必背）：
① 输入井 / 输出井（磁盘上，暂存多个进程的 I/O 数据）
② 输入缓冲区 / 输出缓冲区（内存中，中转）
③ 输入进程 / 输出进程（管理模块，负责调度）
三大特点（必背）：① 提高 I/O 速度 ② 独占设备改造为共享设备 ③ 实现虚拟设备功能。
设备分配四张表
表全称作用
SDT系统设备表总入口，记录系统所有设备
DCB/DCT设备控制块（表）每个设备一张，记录设备状态
COCB/COCT控制器控制块（表）设备连到哪个控制器
CHCB/CHCT通道控制块（表）控制器连到哪个通道
分配路径：SDT → DCB → COCB → CHCB，形成一条从内存到设备的通路。

08手把手演示（大题）

本章大题的两大题型：I/O 完成 5 步骤排序 + SPOOLing 原理说明。下面手把手演示。

演示 1 · I/O 完成 5 步骤排序（大题标准题型）

这是本章最典型的大题：给你一堆语句，让你按 I/O 从请求到完成的正确顺序排列。

场景：请求者进程要读磁盘上某文件的第 N 号记录。涉及请求者进程、磁盘驱动进程、通道、CPU 中断处理。
请把 I/O 完成的 5 个步骤按正确顺序排列，并说清每步谁在干什么。
第 1 步 · 理解角色
参与的有四方：
• 请求者进程（要数据的一方）
• 磁盘驱动进程（管磁盘设备的中间人，出现两次）
• 通道 + 磁盘硬件（实际干活的）
• CPU 中断处理程序（完成后收尾的）
第 2 步 · 5 步骤正确顺序
步骤谁干什么
① 请求请求者进程发出读第 N 号记录的请求，求逻辑块号，阻塞自己等结果
② 启动磁盘驱动进程（第1次）把逻辑块号转物理地址，组织通道程序，启动通道，然后自己阻塞
③ 传送+中断通道+磁盘通道指挥磁盘把数据传到内存缓冲区，完成后向 CPU 发完成中断
④ 中断处理CPU 中断处理程序响应中断，分析原因，唤醒磁盘驱动进程（第2次运行）
⑤ 交付磁盘驱动进程→请求者把缓冲区的数据分离出来传给请求者，唤醒请求者进程
第 3 步 · 记忆口诀
请求 → 启动 → 传送中断 → 中断处理 → 交付
关键：请求者只管提需求和等结果（首尾），中间三步由驱动进程（两次）和 CPU 完成。注意磁盘驱动进程运行了两次（第②步启动、第④步后收尾）。
答：① 请求者发请求 → ② 驱动进程启动通道 → ③ 通道传送并完成后中断 → ④ CPU 中断处理唤醒驱动 → ⑤ 驱动把数据交付请求者。
口诀：请求-启动-中断-处理-交付；请求者首尾出现，驱动进程两次运行。

演示 2 · SPOOLing 原理（以打印机为例）

另一类大题：解释 SPOOLing 如何把独占设备变共享。

场景：多个进程都要用打印机打印。打印机是独占设备，直接分配会死锁或低效。
用 SPOOLing 解决，说明原理。
第 1 步 · 不用 SPOOLing 的问题
打印机一次只能服务一个进程。进程 A 打印时，进程 B、C 只能等。如果 A 申请到打印机后阻塞了不释放，B、C 会死锁。
第 2 步 · SPOOLing 的做法
在磁盘上辟一块"输出井"，每个进程要打印时：
① 进程的打印数据先写到输出井（假装打印完了，进程立即返回干别的事）
② 输出井里排起打印队列
③ SPOOLing 的输出进程从队列里取数据，慢慢送到真正的打印机打印
关键对进程来说，"打印"瞬间完成（其实只是写到磁盘）；真正的打印在后台慢慢做。
第 3 步 · 为什么算"共享"
多个进程的打印请求都进了输出井排队，打印机物理上还是一次打一个，但逻辑上多个进程"同时"在用——这就是"虚拟设备"。
三大特点：① 提高 I/O 速度（进程不用等）② 独占变共享 ③ 实现虚拟设备。
答：SPOOLing 用磁盘输出井做缓冲，进程的打印请求先入井排队，后台进程慢慢送到打印机。物理独占，逻辑共享，实现虚拟设备。

09误解与边界

误解 1：SPOOLing 就是缓冲技术
推论错SPOOLing 用了缓冲（缓冲区是它的一部分），但它本质是"用磁盘把独占设备虚拟成共享设备"，不只是缓冲。缓冲只暂存数据，SPOOLing 还涉及进程管理和设备改造。
误解 2：DMA 不需要任何 CPU 参与
事实错DMA 只是减少了 CPU 参与（中间传数据不用 CPU），但开始时 CPU 要发"传送一块"命令，结束时要做处理。完全不需要 CPU 的是通道方式。
误解 3：SPOOLing 的输入井/输出井在内存
事实错输入井/输出井在磁盘上（容量大、能暂存多个进程的数据）；内存里的是输入缓冲区/输出缓冲区（容量小，做中转）。别搞混。
误解 4：I/O 完成步骤里请求者进程全程参与
大题扣分点请求者进程只参与第 1 步（发请求）和第 5 步（拿结果），中间 3 步（启动、传送、中断处理）由驱动进程和 CPU 完成，请求者在阻塞等待。

10背景知识缺口
