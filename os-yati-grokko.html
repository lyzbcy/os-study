题1SJF 单道批处理 · 3 作业调度

📎 提纲出处：第 110~122 行（含官方答案）｜题型：进程调度算参 ★必考

提纲原题
单道批处理系统，短作业优先（SJF）
单道批处理系统，采用短作业优先调度算法。当第一个作业进入后开始调度，现有 3 个作业，进入和计算时间如下：

作业1：进入 9:00，需计算 60 分钟
作业2：进入 9:10，需计算 45 分钟
作业3：进入 9:15，需计算 25 分钟

求：(1) 每个作业的开始时间、完成时间、周转时间；(2) 平均周转时间。
第 1 步 · 理解 SJF：CPU 空闲时，从"已到达"的作业里挑最短的
SJF 不是全局找最短，而是每个调度时刻看哪些作业已经到达，从中挑计算时间最短的。
9:00 只有作业1到达 → 它最长也得先跑（没别的选）。
第 2 步 · 画时间轴（关键）
• 9:00：作业1到达，CPU 空 → 作业1开始跑
• 9:10：作业2到达，但作业1在跑，作业2等待
• 9:15：作业3到达，作业1在跑，作业3等待
• 10:00：作业1跑完（9:00+60分）。此时等待队列有作业2(45分)和作业3(25分) → SJF 选短的作业3
第 3 步 · 填表
作业号进入时间需计算时间开始时间完成时间周转时间
19:0060 分9:0010:0060 分
39:1525 分10:0010:2570 分
29:1045 分10:2511:10120 分
易错完成顺序是 1→3→2，不是到达顺序 1→2→3！因为作业3比作业2短，作业1完成后作业3先跑。
第 4 步 · 算平均周转时间
平均周转时间 = (60 + 70 + 120) / 3 = 250 / 3 ≈ 83.3 分钟
（官方答案：83 分 20 秒，250÷3=83.33）
答：(1) 见上表，执行顺序 1→3→2；
(2) 平均周转时间 ≈ 83 分 20 秒。
核心：SJF 在每个调度点挑"已到达中最短"的，不是全局最短。
检验：作业1在 10:00 完成后，SJF 下一个调度谁？为什么？

题2两道作业批处理 · 就绪进程平分 CPU

📎 提纲出处：第 208~218 行（含官方答案）｜题型：多道+时间片平分 ★必考

提纲原题
两道作业的批处理系统，FCFS 作业调度 + 内存就绪进程平分 CPU
一个具有两道作业的批处理系统中，作业调度采用先来先服务（FCFS），作业一旦被调度运行就不再退出，作业对应的进程调度采用内存中的就绪进程平分 CPU 时间，不计作业对换及其他系统开销。作业序列：

作业号 | 进入时刻 | 估计运行时间
JOB1 | 8:00 | 25 分钟
JOB2 | 8:20 | 30 分钟
JOB3 | 8:25 | 25 分钟
JOB4 | 8:30 | 15 分钟

求：(1) 各作业开始/结束/周转时间；(2) 平均周转时间、平均带权周转时间。
第 1 步 · 理解"两道"和"平分 CPU"
两道＝内存最多同时放 2 个作业。作业调度 FCFS：作业到达且内存有空位就调入。
平分 CPU＝内存里有几个就绪进程，CPU 就被几个进程平分（相当于轮转，2 个进程各得 50% CPU 速度）。
第 2 步 · 逐时刻调度分析
• 8:00：JOB1 到达，内存空，调入。内存={JOB1}，只有1个进程，CPU 全给 JOB1（速度 100%）。
• 8:20：JOB2 到达，内存有1空位，调入。内存={JOB1,JOB2}，平分 CPU，各得 50%。
　　注意：JOB1 在 8:00~8:20 独占 CPU 跑了 20 分钟（满速），还剩 25−20=5 分钟工作量。
• 8:25：JOB3 到达，但内存满（2道），JOB3 在外存等待。
　　8:20~8:25 这 5 分钟，JOB1、JOB2 各分到 2.5 分钟。JOB1 剩余 5 分钟工作量跑完 → JOB1 在 8:30 完成（5÷0.5=10分，8:20+10分=8:30）。
• 8:30：JOB1 完成，内存空出 1 位；同时 JOB4 到达。FCFS 作业调度 → JOB3 先于 JOB4（JOB3 8:25 到达更早），调入 JOB3。内存={JOB2,JOB3}。
• 此后 JOB2、JOB3 平分 CPU（各 50%），直到有进程完成……（后续类似推演）
第 3 步 · 完整时间表（官方答案）
作业号进入实际运行开始结束周转带权周转
18:0025分8:008:3030分30/25
28:2030分8:209:2060分60/30
38:2525分8:309:2055分55/25
48:3015分9:209:3565分65/15
关键实际运行时间是工作量，但因平分 CPU，墙钟时间（实际经过）= 工作量 ÷ 当前进程数。如 JOB2 从 8:20 开始还有 30 分工作量，前 10 分钟独占(JOB1 还在但即将完成)…复杂处在于进程数随完成事件变化。
第 4 步 · 算平均周转和带权周转
周转总和 = 30 + 60 + 55 + 65 = 210 分
平均周转 = 210 / 4 = 52.5 分
带权周转 = 30/25 + 60/30 + 55/25 + 65/15
= 1.2 + 2.0 + 2.2 + 4.33 ≈ 9.73
平均带权周转 = 9.73 / 4 ≈ 2.43
答：(1) 见上表；(2) 平均周转 52.5 分，平均带权周转 ≈ 2.43（官方答 2.425）。
核心难点：进程数变化导致 CPU 分配比例变化，要分时段算。

题3银行家算法 · 10 台打印机 3 进程

📎 提纲出处：第 506~516 行（含官方答案）｜题型：银行家安全性分析 ★可能考

提纲原题
某系统中有 10 台打印机，3 进程竞争
某系统中有 10 台打印机，有 3 个进程 P1、P2、P3 分别需要 8 台、7 台、4 台。若 P1、P2、P3 已经申请到 4 台、2 台和 2 台。
问：按银行家算法能安全分配吗？请说明分配的过程。
第 1 步 · 算 Need（每个进程还差多少）
Need = Max − Allocation
• P1：8 − 4 = 4
• P2：7 − 2 = 5
• P3：4 − 2 = 2
第 2 步 · 算当前可用 Available
总共 10 台，已分 4+2+2 = 8 台
Available = 10 − 8 = 2 台
第 3 步 · 安全性检查：逐轮找 Need ≤ Work 的进程
设 Work = Available = 2，Finish 全为 false。

第 1 轮（Work=2）：
• P1：Need 4 > 2 ✗
• P2：Need 5 > 2 ✗
• P3：Need 2 ≤ 2 ✓ → P3 跑完，归还它的 2 台。Work = 2+2 = 4；Finish[P3]=true

第 2 轮（Work=4）：
• P1：Need 4 ≤ 4 ✓ → P1 跑完，归还 4 台。Work = 4+4 = 8；Finish[P1]=true

第 3 轮（Work=8）：
• P2：Need 5 ≤ 8 ✓ → P2 跑完，归还 2 台。Work = 8+2 = 10；Finish[P2]=true
第 4 步 · 判定
所有 Finish = true → 系统安全。
安全序列 = <P3, P1, P2>。
答：系统是安全的，安全序列 <P3, P1, P2>。
分配过程：剩余 2 台先给 P3（满足其全部需求）→ P3 完成归还 4 台 → 给 P1 → P1 完成归还 8 台 → 给 P2。每个进程都能完成。
注意：归还的是该进程的 Allocation（已分到的），不是 Need！
检验：安全性算法中，一个进程跑完后，Work（可用资源）应该加上该进程的哪个值？

提纲原题
3 进程 12 台磁带机的安全状态判定
假定系统有 3 个进程 P1、P2、P3，共有 12 台磁带机。进程 P1 总共要求 10 台，P2 和 P3 分别要求 4 台和 9 台。设在 T0 时刻，P1、P2、P3 已经获得 5 台、2 台和 2 台。

第 1 步 · 算 Need
• P1：10 − 5 = 5
• P2：4 − 2 = 2
• P3：9 − 2 = 7
第 2 步 · 算 Available
总共 12 台，已分 5+2+2 = 9 台
Available = 12 − 9 = 3 台
第 3 步 · 安全性检查

第 1 轮（Work=3）：

• P2：Need 2 ≤ 3 ✓ → P2 完成，归还 2 台。Work = 3+2 = 5；Finish[P2]=true

• P1：Need 5 ≤ 5 ✓ → P1 完成，归还 5 台。Work = 5+5 = 10；Finish[P1]=true

• P3：Need 7 ≤ 10 ✓ → P3 完成，归还 2 台。Work=10+2=12；Finish[P3]=true
第 4 步 · 判定
所有 Finish = true → T0 时刻系统安全。
安全序列 = <P2, P1, P3>。
答：T0 时刻系统是安全的，存在安全序列 <P2, P1, P3>。

题5页面置换 · FIFO / LRU（7 页 4 块）

📎 提纲出处：第 666~676 行（含官方答案）｜题型：页面置换画表 ★必考

提纲原题
页式存储管理 · FIFO 与 LRU 缺页对比
某采用页式存储管理的系统，接收了一共 7 页的作业，作业执行时依次访问的页为：
1、2、3、4、2、1、5、6、2、1、2、3、7、6、3、2、1、2、3、6
分别用 FIFO 与 LRU 页面调度算法，在得到 4 块主存空间情况下，分别列出各自的页面淘汰顺序和缺页中断次数，以及最后留驻主存 4 页的顺序。
（假定前面四页 1 2 3 4 已在主存）
第 1 步 · 理解前提："前四页已在主存"
访问序列前 4 个是 1、2、3、4，题目假定它们已在主存——所以这 4 次访问不产生缺页（不调入也不淘汰）。
内存初始状态：{1, 2, 3, 4}。后续从第 5 个访问（页2）开始算缺页。
关键这是最容易算错的点：有些题"从空内存开始"每个新页都缺页，本题"前四页已装入"前四次不算！
第 2 步 · FIFO 推演（淘汰最早进来的）
FIFO 用队列，淘汰队首（最先进入的）。内存初始队列：[1,2,3,4]（1最早）。
• 访问 2：在内存，命中。队列不变 [1,2,3,4]
• 访问 1：在内存，命中。[1,2,3,4]
• 访问 5：不在内存，缺页①！淘汰队首 1 → [2,3,4,5]
• 访问 6：不在，缺页②！淘汰 2 → [3,4,5,6]
• 访问 2：不在，缺页③！淘汰 3 → [4,5,6,2]
• 访问 1：不在，缺页④！淘汰 4 → [5,6,2,1]
• 访问 2：在（队列有2），命中。[5,6,2,1]
• 访问 3：不在，缺页⑤！淘汰 5 → [6,2,1,3]
• 访问 7：不在，缺页⑥！淘汰 6 → [2,1,3,7]
• 访问 6：不在，缺页⑦！淘汰 2 → [1,3,7,6]
• 访问 3：在，命中。[1,3,7,6]
• 访问 2：不在，缺页⑧！淘汰 1 → [3,7,6,2]
• 访问 1：不在，缺页⑨！淘汰 3 → [7,6,2,1]
• 访问 2：在，命中。[7,6,2,1]
• 访问 3：不在，缺页⑩！淘汰 7 → [6,2,1,3]
• 访问 6：在，命中。[6,2,1,3]
淘汰顺序：1, 2, 3, 4, 5, 6, 2, 1, 3, 7 → 共 10 次（官方答案 10 次）
最后驻留：6, 2, 1, 3（官方答 1 3 6 2）
第 3 步 · LRU 推演（淘汰最近最久未用）
LRU 每次淘汰"最久没被访问"的页。用"最近访问时间"排序。
• 访问 2、1：命中，更新它们的"最近访问时间"
• 访问 5：不在，缺页①！此时最久没用的是 3（1和2刚用过）→ 淘汰 3 → {1,2,4,5}
• 访问 6：不在，缺页②！最久没用是 4 → 淘汰 4 → {1,2,5,6}
• 访问 2、1、2：命中
• 访问 3：不在，缺页③！最久没用是 5 → 淘汰 5 → {1,2,6,3}
• 访问 7：不在，缺页④！最久没用是 6 → 淘汰 6 → {1,2,3,7}
• 访问 6：不在，缺页⑤！最久没用是 1 → 淘汰 1 → {2,3,7,6}
• 访问 3、2、1：3、2 命中，1 不在，缺页⑥！最久没用是 7 → 淘汰 7 → {2,3,6,1}
• 访问 2、3、6：全部命中
淘汰顺序：3, 4, 5, 6, 1, 7 → 共 6 次（官方答案 6 次）
最后驻留：6, 3, 2, 1（官方答 6 3 2 1）
第 4 步 · 对比结论
算法缺页次数淘汰顺序最后驻留
FIFO10 次1,2,3,4,5,6,2,1,3,76,2,1,3
LRU6 次3,4,5,6,1,76,3,2,1
LRU 缺页更少（6 vs 10），因为 LRU 基于局部性原理，淘汰真正不用的页；FIFO 盲目淘汰最早的，可能淘汰马上要用的。
答：FIFO 缺页 10 次（淘汰 1,2,3,4,5,6,2,1,3,7），LRU 缺页 6 次（淘汰 3,4,5,6,1,7）。
核心：① 本题"前四页已装入"前4次不缺页；② LRU 看最近访问时间，FIFO 看进入顺序。
检验：为什么这道题 LRU 的缺页次数（6次）比 FIFO（10次）少？

题6页式地址变换 · 64KB / 16 块

📎 提纲出处：第 720~740 行（含官方答案）｜题型：地址变换三步法 ★必考

提纲原题
页式管理 · 求总长度、起始地址、物理地址
假定某页式管理系统，主存为 64KB，分成 16 块，块号为 0、1、2……15。设某作业有 4 页，其页号为 0、1、2、3，被分别装入主存的 2、4、1、6 块。试问：
(1) 该作业的总长度是多少字节？
(2) 写出该作业每一页在主存中的起始地址。
(3) 若给出逻辑地址 [0,100]、[1,50]、[2,0]、[3,60]，请计算相应的内存地址。
第 1 步 · 算每块大小（页大小 = 块大小）
每块大小 = 主存总容量 ÷ 块数 = 64KB ÷ 16 = 4KB
所以页面大小 = 4KB = 4096 字节。
第 2 步 · (1) 算作业总长度
作业有 4 页，每页 4KB：
总长度 = 4 页 × 4KB = 16KB
（官方答案：16KB）
第 3 步 · (2) 算每页的起始地址
每页装入对应块，块的起始地址 = 块号 × 块大小（4KB = 4096B）：
• 页0 → 块2：2 × 4096 = 8192（8K）
• 页1 → 块4：4 × 4096 = 16384（16K）
• 页2 → 块1：1 × 4096 = 4096（4K）
• 页3 → 块6：6 × 4096 = 24576（24K）
第 4 步 · (3) 地址变换三步法
公式：逻辑地址 [页号 P, 页内偏移 W] → 查页表得块号 F → 物理地址 = F × 块大小 + W

• [0, 100]：页号0 → 查表得块号 2。
　　物理地址 = 2 × 4096 + 100 = 8192 + 100 = 8292
• [1, 50]：页号1 → 块号 4。
　　物理地址 = 4 × 4096 + 50 = 16384 + 50 = 16434
• [2, 0]：页号2 → 块号 1。
　　物理地址 = 1 × 4096 + 0 = 4096
• [3, 60]：页号3 → 块号 6。
　　物理地址 = 6 × 4096 + 60 = 24576 + 60 = 24636

第5章提纲里没有计算题，大题方向是简答/分析。两种最高频考法：

(2) 说明程序查询、中断、DMA、通道四种 I/O 控制方式的区别。
SPOOLing 标准答题模板（必背三段）
① 组成（三部分）：
• 输入井 / 输出井——在磁盘上，暂存多个进程的 I/O 数据
• 输入缓冲区 / 输出缓冲区——在内存中，作中转
• 输入进程 / 输出进程——管理模块，负责调度
② 工作原理（以打印机为例）：
用户进程要打印时，数据先写入磁盘的输出井（进程立即返回，假装"打印完了"），输出井排队；SPOOLing 的输出进程利用 CPU 空闲时间，从输出井慢慢送到真正的打印机。物理上打印机一次只打一个，但逻辑上多个进程"同时"在用。
③ 三大特点：提高 I/O 速度、将独占设备改造为共享设备、实现虚拟设备功能。
易错点（阅卷扣分项）
• 输入井/输出井在磁盘，不在内存！内存里的是缓冲区。两者别搞混。
• SPOOLing 本质不是缓冲技术，是用磁盘把独占设备虚拟成共享设备（缓冲只是它的一部分）。
• "虚拟设备"= 物理上独占，逻辑上共享。
四种 I/O 控制方式对比（另一高频简答）
方式谁传数据CPU 参与度并行程度
程序查询CPU全程循环测试最低（CPU外设串行）
中断方式CPU间歇（每字干预）中（CPU外设并行）
DMA 方式DMA控制器少（仅首尾）较高（成块传）
通道方式通道（独立处理机）极少（发指令即可）最高（完全代管）
答题要点：四种方式并行程度递增，本质区别在于"CPU 和外设并行工作的程度不同"。程序查询 CPU 全程忙等；中断解放了 CPU 但每传一个字要干预；DMA 成块传送 CPU 只管首尾；通道是独立处理机完全代替 CPU 管 I/O。

• P2 Need'(0,4,3,0)：B=4 > 2 ✗

题12页面置换 · OPT / LRU / FIFO（M=4）

📎 真题来源：2025 年期末 A 卷 · 第 4 题（原题数据）｜题型：三种置换算法缺页对比 ★必考

2025 真题
请求分页 · 4 物理块 · OPT/LRU/FIFO 缺页次数与缺页率
请求分页系统，作业页面走向：
4, 3, 2, 1, 4, 3, 5, 4, 3, 2, 1, 5
分配物理块数 M = 4（从空内存开始）。
分别用 OPT（最佳置换）、LRU、FIFO 算缺页次数和缺页率。
第 1 步 · 前 4 次访问：4,3,2,1（都缺页，装满内存）
内存从空开始，前 4 个不同页都要调入：
访问 4 → 缺页，内存{4}
访问 3 → 缺页，内存{4,3}
访问 2 → 缺页，内存{4,3,2}
访问 1 → 缺页，内存{4,3,2,1}（满）
已缺页 4 次。三种算法前 4 次完全一样。
第 2 步 · OPT 推演（淘汰"未来最久不再用"的页）
续访问 4,3：命中（内存有）。
访问 5：缺页！内存满。看未来序列 4,3,2,1,5（剩余访问：4,3,2,1,5）：
　• 页4：后面会访问（第7个4）
　• 页3：后面会访问（第8个3）
　• 页2：后面会访问（第9个2）
　• 页1：后面在第10个才访问，最久 → OPT 淘汰 1 → 内存{4,3,2,5}
访问 4,3：命中。访问 2：命中。访问 1：缺页！未来只剩 5，页5马上用、页4/3/2不再出现……看剩余只有"5"，淘汰 2（未来不再用）→ {4,3,1,5}
访问 5：命中。
OPT 缺页 = 4（前4次）+ 1（访问5）+ 1（访问1）= 6 次
缺页率 = 6/12 = 50%
第 3 步 · LRU 推演（淘汰"最近最久未用"）
内存{4,3,2,1}（按最近使用：1最新，4最旧）。
访问 4：命中，4 变最新。访问 3：命中，3 变最新。
访问 5：缺页！最近最久未用是 2（4和3刚用过，1比2近）→ 淘汰 2 → {4,3,1,5}
访问 4：命中。访问 3：命中。访问 2：缺页！最久未用是 1 → 淘汰 1 → {4,3,5,2}
访问 1：缺页！最久未用是 5 → 淘汰 5 → {4,3,2,1}
访问 5：缺页！最久未用是 4 → 淘汰 4 → {3,2,1,5}
LRU 缺页 = 4 + 1(5) + 1(2) + 1(1) + 1(5) = 8 次
缺页率 = 8/12 ≈ 66.7%
第 4 步 · FIFO 推演（淘汰"最早进入"的页，队列）
队列（按进入顺序）：[4,3,2,1]，队首 4 最早。
访问 4,3：命中（注意 FIFO 命中不改队列顺序）。
访问 5：缺页！淘汰队首 4 → [3,2,1,5]
访问 4：缺页！淘汰队首 3 → [2,1,5,4]
访问 3：缺页！淘汰队首 2 → [1,5,4,3]
访问 2：缺页！淘汰队首 1 → [5,4,3,2]
访问 1：缺页！淘汰队首 5 → [4,3,2,1]
访问 5：缺页！淘汰队首 4 → [3,2,1,5]

缺页率 = 10/12 ≈ 83.3%

LRU866.7%
FIFO1083.3%

核心：OPT 看未来最久不用，LRU 看过去最久未用，FIFO 看进入顺序；缺页率 = 缺页次数 ÷ 总访问次数。

题13磁盘调度 · FCFS / SSTF / LOOK

📎 真题来源：2025 年期末 A 卷 · 第 5 题（原题数据）｜题型：磁盘调度 ★第4章必考

2025 真题
200 柱面 · FCFS / SSTF / LOOK 三算法移动量
磁盘 200 柱面（编号 0~199）。完成磁道 125 请求后，当前在 143 号为一请求服务。请求队列顺序：
86, 147, 91, 177, 94, 150, 102, 175, 130
分别用 FCFS、SSTF、LOOK，写出磁头移动顺序、总移动量、平均寻道长度。
第 1 步 · FCFS（按队列顺序服务）
起点 143，按队列 86→147→91→177→94→150→102→175→130：
移动距离逐段：
|86−143|=57，|147−86|=61，|91−147|=56，|177−91|=86，|94−177|=83，|150−94|=56，|102−150|=48，|175−102|=73，|130−175|=45
总移动 = 57+61+56+86+83+56+48+73+45 = 565
平均寻道 = 565 / 9 ≈ 62.8
第 2 步 · SSTF（每次选距当前最近的请求）
从 143 出发，找最近：队列中距 143 最近的是 147（距离4）或 130（距离13）→ 选 147：
143 → 147(4) → 150(3) → 130(20) → 102(28) → 94(8) → 91(3) → 86(5) → 175(89) → 177(2)
移动：4+3+20+28+8+3+5+89+2 = 162
校验SSTF 每步贪心选最近，总移动 162，平均 = 162/9 = 18.0
第 3 步 · LOOK（电梯算法，向当前方向走到"最后一个请求"就反向，不到物理边界）
LOOK 与 SCAN 区别：SCAN 走到磁盘物理边界（0 或 199），LOOK 只走到该方向上最远的请求就折返，省去空跑。
本题未给定初始方向，按惯例假设向增大方向（与题7一致）：
先服务 ≥143 的请求（从小到大）：147, 150, 175, 177 → 到 177（增大方向最远请求）折返
再服务 <143 的请求（从大到小）：130, 102, 94, 91, 86
完整顺序：143 → 147 → 150 → 175 → 177 → 130 → 102 → 94 → 91 → 86
移动：增大段 177−143=34；折返到 86 走 177−86=91
总移动 = 34 + 91 = 125，平均 = 125 / 9 ≈ 13.9
算法总移动量平均寻道
FCFS56562.8
SSTF16218.0
LOOK12513.9
第 4 步 · 三算法对比记忆
• FCFS：公平、实现简单，但总移动最大（565），对随机请求性能差。
• SSTF：贪心选最近，总移动少（162），但可能饥饿（远处请求一直等）。
• LOOK：电梯式扫描，总移动最少（125），无饥饿，是实用最优。
易错LOOK vs SCAN：SCAN 到物理边界（0/199），LOOK 只到最远请求。本题若写 SCAN 会多算 177→199→… 的空跑。
答：FCFS 总移动 565（平均 62.8）；SSTF 162（平均 18.0）；LOOK 125（平均 13.9）。
核心：LOOK 不走到物理边界，只到该方向最远请求即折返，故比 SCAN 更省；三算法中 LOOK 实用最优。

题14PV 操作 · 三进程同步（生产者-消费者 · 奇偶数分类）

📎 真题来源：2025 年期末 A 卷 · 第 6 题（原题数据）｜题型：PV 信号量同步互斥 ★第2章必考

2025 真题
三进程共享 N 格缓冲区 · 奇偶数分别消费
三个进程 P1、P2、P3 互斥使用一个含 N(N>0) 个单元的缓冲区：
• P1：每次 produce() 生成一个正整数，用 put() 放入缓冲区空单元
• P2：每次 getodd() 从缓冲区取出一个奇数，用 countodd() 统计奇数个数
• P3：每次 geteven() 从缓冲区取出一个偶数，用 counteven() 统计偶数个数
用信号量机制实现三进程的同步与互斥，说明信号量含义，用伪代码描述。
第 1 步 · 分析：这是生产者-消费者的变体
经典生产者-消费者只有"一个消费者"，而这题有两个消费者，分别只取奇数和偶数。所以"缓冲区有几类数据可用"要拆成两个计数信号量：
• 缓冲区空位数 → empty（初值 N）
• 缓冲区里奇数个数 → odd（初值 0）
• 缓冲区里偶数个数 → even（初值 0）
• 互斥访问缓冲区（临界区）→ mutex（初值 1）
第 2 步 · 信号量声明与含义
semaphore empty = N; // 缓冲区空单元数，P1 放入前 P，P2/P3 取出后 V semaphore odd = 0; // 缓冲区中奇数个数，P1 放奇数后 V，P2 取奇数前 P semaphore even = 0; // 缓冲区中偶数个数，P1 放偶数后 V，P3 取偶数前 P semaphore mutex = 1; // 互斥信号量，保证对缓冲区的互斥访问
关键empty 和 odd/even 是同步信号量（协调"能不能做"），mutex 是互斥信号量（协调"独占访问"）。
第 3 步 · P1（生产者）伪代码
P1 生成数后，要先申请空位（P(empty)），再互斥放入（P(mutex)…V(mutex)），最后根据奇偶性唤醒对应消费者（V(odd) 或 V(even)）：
process P1() { while(true) { x = produce(); // 生成一个正整数 P(empty); // 申请一个空位（满了就等） P(mutex); // 进入临界区 put(x); // 放入缓冲区 V(mutex); // 离开临界区 if (x % 2 == 1) V(odd); // 是奇数，通知 P2 else V(even); // 是偶数，通知 P3 } }
第 4 步 · P2（取奇数）、P3（取偶数）伪代码
P2 等"有奇数"（P(odd)），再互斥取出（P(mutex)…V(mutex)），最后释放空位（V(empty)）；P3 同理用 even：
process P2() { // 取奇数 while(true) { P(odd); // 等待缓冲区有奇数 P(mutex); // 进入临界区 getodd(); // 取出一个奇数 V(mutex); // 离开临界区 V(empty); // 释放一个空位 countodd(); // 统计（在临界区外，不占缓冲区） } } process P3() { // 取偶数 while(true) { P(even); // 等待缓冲区有偶数 P(mutex); geteven(); V(mutex); V(empty); counteven(); } }
第 5 步 · 易错点（阅卷扣分项）
• P 操作顺序不能错：必须先 P(同步信号量) 再 P(mutex)，不能先 P(mutex)！否则可能死锁（拿了 mutex 却因没数据阻塞，别人永远进不来）。
• odd/even 必须分开：不能只用一个 full 信号量，否则 P2 可能把偶数取走（类型不符）。
• V 操作顺序无所谓：V(mutex) 和 V(empty) 谁先都行，但都要有，不能漏。
• countodd/counteven 放临界区外即可（它们不访问缓冲区）。
答：4 个信号量 empty=N, odd=0, even=0, mutex=1。P1 放数前 P(empty)→P(mutex)→put→V(mutex)→(奇?V(odd):V(even))；P2 取奇 P(odd)→P(mutex)→getodd→V(mutex)→V(empty)；P3 取偶 P(even)→P(mutex)→geteven→V(mutex)→V(empty)。
核心：奇偶分两个同步信号量；同步 P 必须在互斥 P 之前，否则死锁。
